# 데이터베이스 구조 — 부울경 AI기업 관리 시스템

본 시스템은 PostgreSQL 데이터베이스를 사용합니다. 모든 정보는 ‘기업’을 중심으로 연결되어, 한 기업의 협업·실무자·컨택 이력을 통합 조회할 수 있도록 설계했습니다.

---

## 한눈에 보는 테이블 관계

```
[기업기본정보] ──1:1── [협업정보]
       │
       ├──1:N── [실무자정보]
       │             │
       │             └──1:N── [컨택이력]
       │
       └──1:N── [컨택이력]
```

- 1:1 — 기업 하나당 협업정보 한 줄.
- 1:N — 기업 하나에 실무자·컨택 이력은 여러 건 가능.
- 컨택 이력은 ‘어느 실무자와 진행했는지’를 선택적으로 연결할 수 있습니다.

핵심 테이블 4개 + 운영 보조 테이블 2개로 구성됩니다.

## 보조 테이블 (시스템용, 사용자가 직접 다루지 않음)

### `dart_corp_codes` — 자동조회 캐시
금융감독원 전자공시(DART)에서 받은 **약 11만 8천 개 기업의 ‘회사명 ↔ 고유번호’ 매핑**을 미리 저장해 둔 표입니다. ‘이름으로 자동 채움’ 시 매칭 속도를 빠르게 하기 위함이며, 한 달에 한 번 정도 갱신해 줍니다.

### `counters` — 코드 자동 부여
화면에서 보이는 사람이 읽는 코드(`C-001`, `P-0001`, `H-0001`)를 일관되게 발급하기 위한 단순 카운터입니다. 동시에 등록해도 번호가 겹치지 않도록 시스템이 안전하게 관리합니다.

---

## 안전장치 요약

- **동시 편집 보호** — 모든 데이터 수정 시 ‘변경 버전’을 확인해, 늦게 저장하는 사람이 먼저 저장한 내용을 덮어쓰지 않게 막습니다. 충돌이 감지되면 “다른 사용자가 먼저 수정했습니다”라고 알려 줍니다.
- **삭제 대신 비활성** — 기업·실무자를 ‘비활성’으로 두기만 하면, 화면 목록에서는 사라지지만 **데이터는 보존**됩니다. 연결된 실무자·컨택 이력의 관계도 깨지지 않습니다.
- **자동 번호 부여** — 새 기업·실무자·컨택 이력에는 시스템이 자동으로 번호(`C-001`, `P-0001`, `H-0001`)를 매기므로 사용자가 신경 쓰지 않아도 됩니다.

## 자동조회로 채워지는 항목 (참고)

‘이름으로 자동 채움’ 사용 시 아래 표의 외부 공개 출처에서 정보가 들어옵니다. **이미 값이 있는 칸은 절대 덮어쓰지 않고, 비어 있는 칸만 채웁니다.**

| 채워지는 항목 | 출처 |
|---|---|
| 기업 소개 | 위키백과 |
| 소재지 · 홈페이지 | 네이버 |
| 대표자 · 업종 · 설립일 · 매출 규모 | 금융감독원 DART (상장사·외부감사 대상) |
| 평균연봉 · 신입사원 연봉 | 공개 임금 데이터 (지방공기업) |

> 자세한 한계: 부산 지역 AI 스타트업 등 일반 사기업은 어떤 무료 공개 데이터셋에도 정보가 거의 없어, 자동조회만으로는 소재지·홈페이지 정도만 채워지고 나머지는 직접 입력이 필요합니다.
